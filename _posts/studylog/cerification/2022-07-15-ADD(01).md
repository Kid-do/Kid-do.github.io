## 요구사항 확인
### 현행시스템 분석

[현행시스템 분석 순서]<br>
- 향후 개발하고자 하는 `시스템의 개발범위`,`이행방향성` 설정에 도움을 주기위해 파악함<br>

1. 구성/기능/인터페이스<br>: `기간업무`(주업무),지원업무 기술 / 주요기능과 하부기능 / foramt,통신규약,연계유형등 확인<br>
2. 아키텍쳐 및 소프트웨어 구성도 <br>
3. 하드웨어 및 네트워크 파악 순으로 진행함 : 해당 내용의 구성도를 통해 내용 파악을 가능함<br>`이중화:대체제가 있는지`<br>

[현행시스템 분석서 작성 및 검토하기]<br>
1. 각 구성도와 현황도를 바탕으로 자료를 수집한다.<br>
2. 수집한 자료를 분석한다.<br>
3. 결과를 산출물로 작성한다.<br>
4. 산출물에 대한 검토한다.<br><br>

[개발 기술 환경 정의]<br>
- 운영체제 -- DBMS -- `미들웨어` -- 오픈소스<br>
- `RPC` `MOM` `ORB` `DB접속미들웨어` `TPmonitor` `WAS`<br><br>

[개발 기술 환경 정의 과정]<br>
1. 자료수집<br>
2. 조사자료 분석,개발 기술 환경 설정<br>
3. 요구사항 정의서,목표 시스템 구성도를 반영한후에 검토<br><br>

[요구사항 개발 프로세스/확인]<br>
- `요구공학`:요구사항을 정의하고, 문서화하고, 관리하는 프로세스를 의미한다.<br>
- 과정 : `도출`--`분석`--`명세`--`확인`<br><br>


### 요구사항 확인/분석
[요구사항 분석기법]<br>
 요구사항 분류(기능/비기능) -> 개념모델링(UML ,유스케이스 다이어그램)
->요구사항 할당 -> 요구사항 협상 -> 정형분석><br><br>

[UML]
- 객체지향 모델링언어<br>
- 시각적으로 표현<br>
- `정적 모델 다이어그램(클래스)` /` 동적 모델 다이어그램(유스케이스,시퀀스,통신,상태,활동)`<br><br>


[요구사항 확인 기법]<br>
- 요구사항 검토/프로토타이핑/모델 검증/인수테스트<br><br><br>

## 데이터 입출력 구현<br><br>
### 개논물 모델링
정보 구조를 실체와 관계를 중심으로 체계적으로 표현한 기법 : 데이터 모델링<br><br><br>
[데이터 모델링]<br>
1. 개념 모델링 - ER다이어그램<br>
2. 논리 모델링 - 엔티티,속성,관계 설정 / 정규화 진행<br>
3. 물리 모델링 - 테이블,칼럼,키,뷰 구성<br><br>

`[정규화와 반정규화`]<br>
- 각각 몇차 정규형인지 반정규화를 왜 진행하고, 해당 정규화에서 발생하 오류는 무엇인지<br><br>

`[오브젝트 설계]`<br>
1. 참조무결성 : 관련 있는 테이블 사이에 참조하는 외래키값은 반드시 기본키로 존재해야한다.<br>
2. 인덱스 / 뷰 설계<br>
3. 클러스터 설계<br>
4. 파티션 / 디스크 구성 설계<br><br>

### 프로시저 작성과 최적화<br>
[데이터 조작 프로시저 작성]<br>
1. 데이터 정의어(DDL)<br>
- CREATE,DROP,RENAME,ALTER,TRUNCATE<br>
2. 데이터 조작어(DML)<br>
-INSERT,UPDATE,DELETE<br>
3. 데이터 제어어(DCL)<br>
- ROLE,GRANT,REVOKE<br>
4. 트렌젝션 제어어(TCL)<br>
- COMMIT, ROLLBACK, SAVEPOINT<br>
5. PL/SQL<br><br>

[데이터 조작 프로시저 최적화]<br>
1. SQL 문제식별(`APM`:식별도구)<br>
2. `옵티마이저` 통계 확인<br>
3. 실행계획검토<br>
4. SQL문 재구성 <br>
5. 실행계획 유지관리<br>
